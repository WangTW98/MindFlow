# Deliverable handoff

Create `deliverable_spec.json` from the validated analysis packet. Include product scope, audiences, journeys, screen/state inventory, navigation and interaction matrix, component/features, data bindings, permissions, responsive/accessibility constraints, acceptance evidence, unresolved decisions, and source/entity traceability.

For a PRD, structure the result around context, goals, scope/exclusions, users/roles, information architecture, functional requirements, flows/states/exceptions, permissions/data, non-functional constraints, acceptance criteria, and unresolved decisions.

For HTML, Figma, or Pencil, hand the same specification to the target-specific capability. Do not invent screens or interaction details just to make the artifact visually complete. Record unsupported visual choices as design assumptions.
