# 多应用端采购协同平台需求示例

## 背景

公司计划建设一套采购协同平台，覆盖管理后台、供应商门户、移动审批 App、公开采购网站四种应用端。平台需要支持采购计划、询价比价、供应商投标、内部审批、合同归档和履约跟踪。不同应用端面向不同用户角色，但业务流程之间存在跨应用端跳转与状态联动。

## 应用端

平台默认需要生成四个独立应用端，每个应用端在画布上都必须有自己的起始卡片节点。跨应用端连线只表示业务流转，不应取消目标应用端自身入口节点的起始属性。

### 管理后台

使用角色：采购专员、采购经理、财务审核员、系统管理员。

业务域：采购计划、询价比价、合同归档、系统管理。

入口页面：采购工作台。采购专员进入后可以查看待处理采购计划、进行询价、查看供应商报价、发起审批和查看合同归档。

### 供应商门户

使用角色：供应商销售、供应商管理员。

业务域：供应商投标、报价协同、合同确认。

入口页面：供应商门户首页。供应商可查看公开或定向询价，提交报价材料，查看中标结果，并确认合同条款。

### 移动审批 App

使用角色：采购经理、财务审核员、总经理。

业务域：移动审批、风险复核。

入口页面：移动审批待办。审批人可以查看采购申请、金额、预算占用、供应商评分、风险提示，并执行通过、退回或转交。

### 公开采购网站

使用角色：访客供应商、潜在供应商。

业务域：公开采购公告、供应商注册。

入口页面：采购公告列表。访客可筛选公告、查看公告详情、提交供应商注册申请。

## 核心业务流程

1. 采购专员在管理后台的采购工作台查看需求池，进入采购计划新建页。
2. 采购计划新建页包含基础信息分组，字段包括计划编号、采购品类、预算金额、期望到货日期、需求部门；还包含附件分组，支持上传需求说明书和预算文件。
3. 采购专员保存草稿后进入采购计划详情页；提交后进入询价方案编辑页。
4. 询价方案编辑页包含供应商筛选分组，字段包括供应商等级、历史履约评分、区域、是否黑名单；包含询价明细分组，字段包括物料名称、规格、数量、目标价格；包含操作按钮：保存方案、发布询价、取消。
5. 发布询价后，管理后台生成询价记录，并向供应商门户推送定向询价任务。
6. 供应商销售在供应商门户首页进入报价填写页。报价填写页包含报价表单分组，字段包括报价金额、税率、交付周期、联系人；包含材料上传分组，支持上传资质文件和报价附件；包含提交报价和撤回报价按钮。
7. 供应商提交报价后，采购专员在管理后台的报价对比页查看所有供应商报价。报价对比页包含筛选表单分组，字段包括供应商名称、报价区间、交付周期、评分；包含结果列表分组，表头包括供应商、报价、税率、交期、评分、风险标签；包含操作按钮：查询、重置、生成比价报告。
8. 生成比价报告后进入审批发起页。审批发起页包含审批摘要分组，字段包括推荐供应商、推荐理由、预算差异、风险提示；包含审批流分组，可选择采购经理、财务审核员和总经理。
9. 审批发起后，移动审批 App 的移动审批待办出现任务。审批详情页展示采购摘要、报价对比、预算占用、风险提示和附件，支持通过、退回、转交。
10. 审批通过后，管理后台进入合同生成页。合同生成页包含合同基础信息分组，字段包括合同编号、合同金额、付款方式、履约周期；包含条款编辑分组，支持编辑付款条款、违约责任、验收标准；包含生成合同和发送供应商确认按钮。
11. 供应商管理员在供应商门户的合同确认页查看合同条款，支持确认、申请修改和下载合同。
12. 双方确认后，合同归档页展示合同状态、签署文件、履约计划、付款节点和操作记录。
13. 公开采购网站的采购公告列表支持访客筛选公告，公告详情页展示采购品类、预算范围、报名截止时间和报名条件。潜在供应商可进入供应商注册页提交企业信息、联系人、资质文件。
14. 系统管理员在管理后台的供应商审核页处理公开网站提交的注册申请，审核通过后供应商可登录供应商门户。

## 画布与节点编辑要求

- 默认页面卡片的连线入口位于卡片左侧标题位置，表示其他卡片可以进入该页面。
- 页面卡片、功能分组、功能项都可以作为连线出口；功能分组出口显示在分组标题右侧，功能项出口显示在功能项标题右侧。
- 连线入口和出口需要使用不同颜色区分，入口用于接收连线，出口用于发起连线。
- 功能分组与功能项必须保持父子关系展示，不允许在详情栏里只显示原始 JSON 字段。
- 详情栏可以新建、编辑、删除功能分组和功能项；功能项可以拖拽到其他功能分组，也可以在同一分组内重新排序。
- 选中应用端、业务域或角色筛选后，在画布空白位置右键新建的页面卡片应继承当前筛选条件。

## 连线示例

- 采购工作台的“新建计划按钮”出口连接到采购计划新建页。
- 询价方案编辑页的“发布询价按钮”出口连接到供应商门户首页，也可连接到报价填写页，用于验证同一个出口节点连接多个目标卡片节点。
- 报价对比页的“生成比价报告按钮”出口连接到审批发起页，同时连接到采购计划新建页用于回看计划，验证一出多目标。
- 移动审批详情页的“退回按钮”出口连接到询价方案编辑页，“通过按钮”出口连接到合同生成页。

## 插件侧栏与本地编辑要求

- VS Code 侧栏需要显示 MindFlow 图标，点击后展示新建 MindFlow、打开 MindFlow 和历史 ProductFlow JSON 列表。
- 卡片详情栏需要支持本地编辑页面标题、页面类型、目的说明、应用端、业务域、角色、输入输出、权限和功能分组。
- 项目概述卡片需要支持本地编辑标题、项目综述、项目目标，以及应用端、业务域、角色和状态组。
- ProductFlow JSON 的可视化修改必须进入 VSCode 文档缓冲区，用户通过文件保存落盘，并可通过编辑撤销/恢复回退。

## 业务域

- 采购计划：需求池、计划新建、计划详情、附件材料。
- 询价比价：询价方案、供应商邀请、报价对比、比价报告。
- 供应商投标：门户首页、报价填写、报价撤回、合同确认。
- 移动审批：待办列表、审批详情、通过、退回、转交。
- 合同归档：合同生成、供应商确认、归档详情、履约节点。
- 公开采购公告：公告列表、公告详情、供应商注册。
- 系统管理：供应商审核、角色权限、黑名单维护。

## 用户角色

- 采购专员：创建采购计划、发布询价、对比报价、发起审批。
- 采购经理：审批采购方案，查看预算和供应商风险。
- 财务审核员：审核预算占用、付款条款和财务风险。
- 总经理：审批高金额采购。
- 供应商销售：查看询价、提交报价和撤回报价。
- 供应商管理员：管理供应商资料、确认合同。
- 访客供应商：浏览公开公告并提交注册。
- 系统管理员：审核供应商注册、维护供应商黑名单和权限。

## 非功能要求

- 每个页面、功能分组、功能项和业务连线需要拥有稳定唯一标识。
- 管理后台、供应商门户、移动审批 App、公开采购网站默认统一展示在同一张流程图中。
- 支持按应用端、业务域和角色筛选，筛选外的节点和连线置灰但不隐藏。
- 支持从功能分组或功能项发起连线，例如“发布询价按钮”连接到供应商门户的“报价填写页”。
- 支持同一个出口连接多个目标页面，例如“生成比价报告”可进入审批发起页，也可进入采购计划详情页。
